# SQL Server Release Guide

## Local Development

Prerequisites:

- .NET 10 SDK.
- SQL Server LocalDB or SQL Server.
- `dotnet-ef` restored from `dotnet-tools.json`.

Commands:

```powershell
dotnet tool restore
dotnet restore
dotnet ef database update
dotnet run
```

Default development connection:

```text
Server=(localdb)\MSSQLLocalDB;Database=N4WTT_Pharmacy;Trusted_Connection=True;MultipleActiveResultSets=true;TrustServerCertificate=True
```

The application runs EF migrations at startup and seeds reference data only when the `Drugs` table is empty.

## SQL Script

`database/schema.sql` is an idempotent migration script generated by:

```powershell
dotnet ef migrations script --idempotent --output database/schema.sql
```

## Publish Package

```powershell
dotnet publish .\Nhom4WebThuocThayThe.csproj -c Release -o .\artifacts\publish
```

Set the production connection string with an environment variable:

```text
ConnectionStrings__PharmacyDatabase=<production SQL Server connection string>
```

## Container

The repository includes a multi-stage `Dockerfile`. The container requires an external SQL Server; LocalDB is Windows-only and is not available inside the Linux runtime image.

```powershell
docker build -t n4wtt-web .
docker run -p 8080:8080 --env-file .env n4wtt-web
```

Health check:

```text
GET /health
```

Expected response:

```json
{"status":"healthy","database":"connected"}
```

## Release Checklist

- Apply database migration.
- Verify `/health`.
- Run acceptance, security and performance suites.
- Verify Admin, Pharmacist, Expert and User access.
- Download a JSON backup before production data changes.
- Record release commit and Jira Sprint Review.
